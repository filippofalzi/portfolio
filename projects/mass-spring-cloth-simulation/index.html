Mass-Spring and Cloth Simulation

A deformable-body simulation developed in Unity, beginning with a general three-dimensional mass-spring system and later extending it into a cloth model generated directly from a triangle mesh.

Technologies Unity, C#, Math.NET Numerics
Techniques Mass-spring systems, force Jacobians and numerical integration
Simulation Elastic forces, damping, fixed regions and deformable meshes

Project overview

The project was developed in two connected stages. The first stage implemented a general mass-spring physics system and compared several numerical integration methods.

The second stage reused the same physical model to simulate a deformable cloth. Mesh vertices became particles, triangle connections generated structural springs and opposite vertices created bending constraints.

Real-time cloth simulation showing deformation, elastic recovery, damping and fixed regions.

1. Mass-spring physical model

The basic model is formed by particles connected through elastic springs. Each node stores its position, velocity, mass and fixed state, while each spring applies forces between two connected nodes.

PARTICLES

Mass nodes

Every node contributes three degrees of freedom and stores its current position, velocity and mass.

CONSTRAINTS

Elastic springs

Springs apply forces according to their stiffness, current length and original rest length.

EXTERNAL FORCES

Gravity and damping

Gravity acts on every free particle, while damping reduces persistent oscillation and numerical instability.

BOUNDARY CONDITIONS

Fixed nodes

Selected particles can be constrained by modifying the system vectors and matrices before each simulation step.

2. Numerical integration

The initial version implements five numerical integration methods. Each method updates the multidimensional position and velocity vectors using a different approximation of the system evolution.

METHOD 01

Explicit Euler

Positions and velocities are updated using forces evaluated at the beginning of the time step.

METHOD 02

Symplectic Euler

Velocity is updated before position, improving stability for many mechanical systems.

METHOD 03

Midpoint

An intermediate state is evaluated halfway through the time step to obtain a more accurate final update.

METHOD 04

Verlet

The new position is calculated using current and previous states, producing stable motion in particle systems.

METHOD 05

Implicit Euler

The future velocity is obtained by solving a linear system that includes force Jacobians.

3. Cloth generation from a mesh

In the cloth extension, the system is built automatically from a Unity triangle mesh.

Unique mesh vertices are converted into mass nodes. Triangle edges generate stretch springs, while vertices located on opposite sides of adjacent triangles are connected through bend springs.

STRETCH

Structural springs

Springs placed along mesh edges preserve the local distance between neighbouring particles.

BEND

Bending springs

Additional springs between neighbouring triangles resist excessive folding and control the flexibility of the cloth.

MESH UPDATE

Real-time deformation

After every physics step, simulated node positions are transformed back into local mesh coordinates and sent to the Unity mesh.

FIXERS

Fixed regions

Collider volumes define regions whose particles remain fixed during the simulation.

4. Force Jacobians and implicit integration

Stiff springs can make explicit methods unstable when the time step is too large. The cloth extension therefore includes position and velocity Jacobians for the elastic and damping forces.

These matrices are used by the implicit solver to compute a more stable future velocity.

Implicit system

A = M - hD - h²K
b = (M - hD)v + hf
Avₙ₊₁ = b

Here, M is the mass matrix, K is the position Jacobian, D is the velocity Jacobian, h is the time step and f is the force vector.

5. Damping and stability

Two damping parameters are used in the cloth model. Particle damping reduces global motion, while spring damping acts along the spring direction according to the relative velocity of connected nodes.

ALPHA DAMPING

Particle damping

A velocity-proportional force is applied independently to every mass node.

BETA DAMPING

Spring damping

Relative motion between connected particles is damped along the current spring direction.

Development progression

The first stage focused on constructing the numerical simulation framework and comparing integration methods. The second stage applied the same principles to a complete deformable surface, adding automatic mesh conversion, structural constraints, damping, Jacobians and fixed regions.

Source code

Both stages are included separately to show the progression from a general mass-spring simulator to the final cloth implementation.
