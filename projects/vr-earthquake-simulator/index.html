Interactive VR Earthquake Simulator

A virtual reality prototype in which the physical movement of an Oculus controller generates a simulated earthquake inside an interactive room.

Technologies Unity, C#, Meta XR SDK and XR Interaction Toolkit
Interaction Controller velocity, grip input and haptic feedback
Simulation Spring-damper dynamics and rigid body response

Project overview

The project explores how movement captured by a virtual reality controller can be converted into a dynamic physical event.

When the user grabs and shakes the controller, its velocity is measured and transformed into an external force applied to the virtual room.

The room oscillates according to a spring-damper model, while furniture and loose objects respond through Unity rigid body physics.

Interactive demonstration showing the controller input, room vibration, object response and Richter-scale feedback.

1. XR controller interaction

The interaction begins when the user grabs the virtual Oculus controller using the grip input.

The controller is configured as an XR Grab Interactable, allowing the user to pick it up, hold it and move it naturally inside the scene.

GRAB INTERACTION

XR Grab Interactable

The controller can be selected and held using the standard XR Interaction Toolkit workflow.

HAND ANIMATION

Grip and trigger input

Grip and trigger values are connected to the hand animator to reproduce the movement of the virtual fingers.

ATTACHMENT

Relative hand alignment

A dedicated attach transform aligns the virtual hand with the controller to improve the apparent grip position.

VISUAL FEEDBACK

Interaction state

Interactive elements change appearance to provide immediate visual feedback during hover and selection.

2. Controller velocity input

The physical controller velocity is read from an Input Action configured as a Vector3 value.

When a physical controller is not available, the XR Simulator uses the difference between consecutive controller positions to estimate velocity.

Simulator fallback

handVelocity = (currentPosition - previousPosition) / deltaTime

This fallback makes it possible to test the complete interaction using mouse and keyboard input inside the Unity editor.

3. Earthquake dynamics

The room is modelled as a forced and damped oscillator. Controller velocity produces the external force that moves the system away from its equilibrium position.

Spring-damper model

m · x'' + c · x' + k · x = F(t)

The stiffness term restores the room towards its original position, while damping dissipates energy and gradually stops the oscillation.

STEP 01

Read velocity

Controller movement is captured as a three-dimensional velocity vector.

STEP 02

Generate force

The velocity is multiplied by an intensity factor to obtain the external seismic force.

STEP 03

Integrate motion

Acceleration, velocity and displacement are updated numerically every frame.

STEP 04

Move the room

The final displacement is applied to the complete room container.

4. Physical object response

Furniture and loose objects inside the room use Unity Rigidbody components.

As the room moves, objects vibrate, slide and fall according to the simulated acceleration and collision response.

RIGID BODY PHYSICS

Dynamic furniture

Objects react independently to the movement of the room through mass, gravity and collisions.

COLLISION DETECTION

Continuous collisions

Continuous collision detection helps prevent fast-moving objects from passing through the floor.

INTERPOLATION

Smoother movement

Rigidbody interpolation reduces visible differences between the physics update and the headset refresh rate.

REAL-TIME RESPONSE

No prerecorded animation

The final motion depends directly on controller input and the physical configuration of the scene.

5. Multisensory feedback

The project combines visual, auditory and haptic feedback to communicate earthquake intensity.

USER INTERFACE

Richter-scale display

A screen-space interface displays a smoothed intensity value that changes colour at high magnitudes.

LIGHTING

Dynamic flickering

Ceiling-light intensity changes in proportion to the seismic input.

SPATIAL AUDIO

Earthquake rumble

Volume is modulated in real time according to the intensity of the controller movement.

HAPTICS

Controller vibration

Haptic impulses are generated when the measured earthquake intensity exceeds a threshold.

6. VR comfort vignette

Rapid movement of the room can increase discomfort in virtual reality.

To reduce this effect, a custom vignette progressively narrows the visible field of view when the simulated earthquake becomes stronger.

Dynamic aperture

aperture = lerp(1.0, 0.25, earthquakeIntensity)

The aperture is smoothed over time and passed to a custom material based on a Fresnel-style visual effect.

Technical value

The project combines XR interaction, controller input, numerical integration, rigid body physics, user-interface feedback, spatial audio and VR comfort design inside a single interactive Unity prototype.
