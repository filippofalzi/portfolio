Co-Rotational Finite Element Elasticity

A deformable-solid simulation developed in Unity using tetrahedral finite elements, linear elasticity, co-rotational deformation and implicit numerical integration.

Technologies Unity, C#, Math.NET Numerics
Simulation model Tetrahedral FEM and continuum elasticity
Numerical methods Symplectic Euler, implicit Euler and sparse matrices

Project overview

This project implements a three-dimensional elastic body using the Finite Element Method. The object is discretized into tetrahedral elements, each of which contributes elastic forces and stiffness terms to the global simulation system.

Unlike a mass-spring model, the FEM formulation represents the deformation of the complete material volume. The simulation therefore evaluates deformation, strain and elastic response inside every tetrahedron.

Implicit FEM simulation showing a tetrahedral beam deforming under load while preserving stable elastic behaviour.

1. Tetrahedral finite element model

The simulated body is divided into tetrahedral elements. Each element contains four nodes with positions in the undeformed and deformed configurations.

Local tetrahedral contributions are calculated independently and later assembled into the global force vector and stiffness matrix.

REST CONFIGURATION

Undeformed geometry

Each tetrahedron stores the original node positions used to define its reference shape and initial volume.

WORLD CONFIGURATION

Deformed geometry

The current node positions describe how each tetrahedral element has moved, stretched, compressed or rotated.

LOCAL COMPUTATION

Element forces

Every element calculates an elastic force contribution for each of its four nodes.

GLOBAL ASSEMBLY

Complete deformable body

Local element vectors and matrices are accumulated into the global simulation system.

2. Deformation gradient

The deformation gradient describes the transformation from the undeformed tetrahedron to its current configuration.

It contains information about stretching, compression, shear and rotation inside the element.

Element deformation

F = Ds · Dm⁻¹

Dm represents the tetrahedron in its rest configuration, while Ds represents its current deformed configuration.

3. Strain and linear elasticity

Strain measures the actual shape deformation of the material. In the linear formulation, the Cauchy strain tensor is calculated from the deformation gradient.

Linear strain

ε = 1/2 (F + Fᵀ) - I

The resulting strain is converted into stress according to the selected elastic material parameters.

STEP 01

Node matrix

Current tetrahedron positions are collected into a local matrix.

STEP 02

Deformation gradient

The rest and deformed tetrahedra are compared through the gradient matrix.

STEP 03

Strain

Local material deformation is measured from the gradient.

STEP 04

Elastic stress

The material model converts strain into an internal stress.

STEP 05

Nodal forces

Stress is converted into forces acting on the tetrahedron nodes.

4. Co-rotational elasticity

A purely linear model incorrectly treats large rigid rotations as material deformation.

The co-rotational formulation separates the rotational component from the actual elastic deformation. The material can therefore rotate while maintaining a physically meaningful strain measurement.

Rotation extraction

F = R · S
Fcorotated = Rᵀ · F

R represents the rigid rotation, while the remaining component contains the elastic deformation used to calculate strain and force.

5. Force Jacobian

The derivative of the elastic force with respect to node positions forms the local force Jacobian.

This matrix describes how forces change when the tetrahedral nodes move. It is required by the implicit solver and contributes to the global stiffness matrix.

LOCAL MATRIX

Tetrahedral stiffness

Each tetrahedron produces a local matrix relating node displacement to elastic force variation.

GLOBAL SYSTEM

Sparse assembly

Local Jacobians are inserted into a sparse global matrix using the indices of the tetrahedron nodes.

6. Symplectic and implicit integration

The simulator supports both symplectic and implicit integration. Symplectic Euler is simpler and less expensive, but it may become unstable for stiff materials or large time steps.

Implicit Euler includes the force Jacobian in the update system, providing greater stability for elastic solids.

Method Main characteristic Typical behaviour
Symplectic Euler Uses current forces to update velocity before updating position. Efficient, but sensitive to stiffness and time-step size.
Implicit Euler Solves a linear system containing the force Jacobian. More computationally expensive, but significantly more stable.

Implicit update

A = M - h² (dF/dx)
b = Mvₙ + hF
Avₙ₊₁ = b

M is the mass matrix, h is the time step, F is the global force vector and dF/dx is the assembled force Jacobian.

7. Sparse matrix assembly

Every tetrahedron affects only the degrees of freedom of its four nodes. The resulting global Jacobian is therefore sparse.

Instead of storing a complete dense matrix, the implementation adds only the non-zero local contributions using sparse matrix triplets.

MEMORY

Sparse storage

Only non-zero matrix coefficients are stored, reducing memory consumption.

PERFORMANCE

Efficient assembly

Each tetrahedron contributes a small local block to the global system.

Technical progression

This project extends deformable-body simulation from discrete mass-spring models to continuum mechanics. The final implementation combines tetrahedral discretization, deformation gradients, strain, co-rotational elasticity, force Jacobians and sparse implicit integration.
