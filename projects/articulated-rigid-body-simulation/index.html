Articulated Rigid Body Simulation

A rigid-body dynamics project developed in Unity, implementing articulated systems through point constraints, numerical integration and both weak and strong constraint formulations.

Technologies Unity, C#, Math.NET Numerics
Simulation model Rigid bodies, inertia tensors and point constraints
Solvers Symplectic Euler, constrained dynamics and implicit Euler

Project overview

This project implements an articulated rigid-body simulator in which multiple bodies can be connected through point constraints.

Each rigid body has six degrees of freedom: three for translation and three for rotation. Linear and angular velocities are integrated while gravity, damping, inertia and constraint forces determine the motion.

Three different simulation approaches were implemented: symplectic integration with weak constraints, symplectic integration with strong constraints and implicit integration with force Jacobians.

Real-time articulated rigid-body simulation showing gravity, rotational motion and point constraints between connected bodies.

1. Rigid-body model

Each simulated object stores its centre of mass, orientation, linear velocity and angular velocity.

Rotation is represented using quaternions, while the inertia tensor is transformed from local coordinates to world coordinates during the simulation.

TRANSLATION

Linear motion

Forces update the linear velocity and centre-of-mass position of each rigid body.

ROTATION

Angular motion

Torques update angular velocity, while incremental quaternion rotations update the body orientation.

MASS PROPERTIES

Inertia tensor

The local inertia tensor is transformed according to the current body orientation before calculating angular acceleration.

EXTERNAL FORCES

Gravity and damping

Gravity affects linear motion, while damping is applied to both linear and angular velocities.

2. Point constraints

Point constraints connect two local points belonging to separate rigid bodies. A constraint can also connect one body to a fixed point in world space.

The simulator transforms local attachment points into world coordinates and evaluates the positional difference between them.

Constraint function

C(q) = pA(qA) - pB(qB)

The constraint is satisfied when the two attachment points occupy the same world-space position.

BODY TO BODY

Articulated connection

Two rigid bodies share a common attachment point while remaining free to rotate around it.

BODY TO WORLD

Fixed attachment

One side of the constraint can remain fixed in world space, allowing chains and pendulum-like structures to be created.

3. Weak constraints

In the weak formulation, a point constraint is treated as a penalty force. The greater the separation between the two attachment points, the greater the restoring force.

Penalty force

Fc = -k Jᵀ C(q)

Here, k is the constraint stiffness, C is the positional error and J is the constraint Jacobian. The transposed Jacobian distributes the correction as forces and torques over the connected bodies.

This approach is easy to integrate with the global force vector, but the constraint can stretch slightly depending on stiffness and time step.

4. Strong constraints

The strong formulation does not add an explicit penalty force. Instead, the body velocities and constraint forces are solved simultaneously.

The global system combines the mass matrix with the constraint Jacobian, producing velocities that directly satisfy the positional constraint.

Constrained linear system

| M Jᵀ | | vₙ₊₁ | = | Mvₙ + hf |
| J 0 | | λ | | -C / h |

The vector λ represents the constraint impulses required to maintain the articulated connection.

5. Implicit integration

The implicit solver evaluates how forces vary with position and velocity. This requires force Jacobians for both rigid bodies and point constraints.

The rigid-body Jacobian includes linear damping, angular damping and the derivative of the gyroscopic torque generated by the inertia tensor.

Implicit system

A = M - h(dF/dv) - h²(dF/dx)
b = [M - h(dF/dv)]vₙ + hf
Avₙ₊₁ = b

The resulting linear system is solved to obtain the new linear and angular velocities.

6. Solver comparison

Method Constraint treatment Main characteristic
Symplectic Euler Weak penalty forces Simple and efficient, but the connection can stretch when the stiffness or time step is not properly adjusted.
Symplectic constrained Strong constraints Solves body velocities and constraint impulses together, maintaining attachment points more accurately.
Implicit Euler Weak constraints with Jacobians Requires a larger linear system but provides greater stability for stiff forces and larger time steps.

Implementation progression

The project moves from basic force integration to a complete constrained rigid-body system. The final implementation combines translational and rotational dynamics, inertia transformation, constraint Jacobians and linear-system solvers within a single Unity simulation framework.
